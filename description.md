Необходимо создать desktop приложение для рисования и анализа графиков.

## Стек технологий:
React (Vite, TypeScript) + uPlot (отрисовка графиков) + PapaParse (парсинг CSV).
Запуск через `npm run dev`, без сборки под конкретную ОС.

## Данные:
Данные для графиков это замеры с датчиков: по оси X время от начала замеров, по оси Y одна из измеряемых величин.
Данные будут поставляться в приложение в формате csv (разделитель — запятая, заголовки в первой строке).
Нужно заложиться на возможность загружать несколько csv файлов.
Предполагается, что пользователь передаст csv файл, в котором одна из колонок будет называться Time и пользователь должен гарантировать, что все значения в этой колонке упорядочены и уникальны, т.е. не будет двух точек с одинаковым значением по оси X. Это необходимо проверять на этапе чтения csv файла, в случае фейла валидации оповещать пользователя окошком.
Ожидается, что значений на каждом графике будет много (в среднем по 100000 точек для каждой величины).

### Несколько CSV файлов:
- CSV файлы могут содержать точки с разными значениями на временной шкале. Шаг по времени может отличаться.
- Пользователь гарантирует, что начало замеров синхронизировано (сдвигать/выравнивать временные шкалы не нужно).
- Графики из всех CSV отображаются на одной координатной плоскости (общий таймлайн). Интерполяция значений между файлами не требуется.

### Именование шкал:
- Название шкалы соответствует названию столбца в CSV.
- Если название столбца в новом CSV совпадает с уже загруженным, к названию новой шкалы добавляется суффикс `_<порядковый_номер>` (например, `Pressure_2`).

## Отрисовка и анализ графиков
1. Измеряемых величин может быть много (как минимум нужно заложиться на 10 штук).
2. Все графики должны отражаться на одной координатной плоскости, при этом у каждого графика должна быть своя шкала по Y. Все Y-шкалы располагаются слева.
3. Графики рисуются прямыми линиями (linear interpolation между точками).
4. Для каждой шкалы (соответствующей своей измеряемой величине) по оси Y:
- дефолтный диапазон значений выбирается от минимального до максимального значения
- на шкале должны быть отражены минимальное значение, максимальное значение, а также INTERMEDIATE_POINTS_CNT промежуточных значений (по умолчанию INTERMEDIATE_POINTS_CNT = 8, нужно сделать это значение настраиваемой константой, для начала достаточно захардкодить)
- пользователю нужно предоставить возможность выбора диапазона значений. Для этого предлагается минимальное и максимальное значение на шкале сделать кликабельным: по нажатию на min или max пользователь может ввести свой вариант значения, после чего график соответствующий данной величине должен быть перерисован в соответствии с новым диапазоном
- нужна кнопка сброса, которая возвращает диапазон данной величины к [min, max]
5. Для оси X:
- дефолтный диапазон значений выбирается от минимального до максимального значения в колонке Time
- на шкале должны быть отражены минимальное значение, максимальное значение, а также INTERMEDIATE_POINTS_CNT промежуточных значений
- пользователю нужно предоставить возможность выбора диапазона значений. Для этого предлагается минимальное и максимальное значение на шкале сделать кликабельным: по нажатию на min или max пользователь может ввести свой вариант значения, после чего графики всех величин должны быть перерисованы в соответствии с новым диапазоном
- нужна кнопка сброса, которая возвращает диапазон Time к [min, max]
- если по оси X выбран суженный диапазон, линии графиков должны рисоваться от ближайших точек, лежащих за границами диапазона, чтобы не было оборванных линий
6. При нажатии ПКМ показывать под мышкой табличку-ховер со значениями всех величин, линейно интерполированными к моменту времени под курсором. Логика тултипа должна быть легко заменяемой/конфигурируемой.
7. Легенда с чекбоксами для показа/скрытия отдельных графиков.

## Сохранение и восстановление сессии

Гибридный подход: авто-сохранение + экспорт/импорт файла.

### Авто-сохранение (IndexedDB):
- Состояние приложения автоматически сохраняется в IndexedDB при каждом значимом действии (загрузка CSV, изменение диапазонов, скрытие серий и т.д.).
- При открытии приложения, если в IndexedDB есть сохранённая сессия — предлагать пользователю «Продолжить предыдущую сессию» или «Начать заново».

### Экспорт/импорт сессии (.dviz):
Файл `.dviz` — это ZIP-архив, содержащий:
- `data.csv` — единый CSV-файл со всеми данными из всех загруженных CSV (объединённый).
- `session.json` — настройки сессии:
  - Путь к файлу данных внутри архива (`data.csv`).
  - Диапазоны осей (X и каждой Y-шкалы).
  - Видимость серий.
  - Цвета линий.
  - Текущая тема.
  - Любые другие пользовательские настройки.

При импорте `.dviz` файла приложение распаковывает архив, загружает данные из CSV и восстанавливает настройки из JSON.

## Внешний вид
- Поддержка светлой и тёмной темы с переключателем.
- Цвета линий — из стандартной палитры, подходящей под обе темы. В будущем предусмотреть возможность пользовательского выбора цвета для каждой шкалы.
