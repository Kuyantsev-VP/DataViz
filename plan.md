# План реализации DataViz

## ~~Шаг 1: Инициализация проекта~~ DONE
- Создать проект через `npm create vite@latest` с шаблоном `react-ts`.
- Установить зависимости: `uplot`, `papaparse`, `@types/papaparse`, `jszip`, `idb-keyval`.
- Настроить базовую структуру папок:
  ```
  src/
    components/    — React-компоненты
    hooks/         — кастомные хуки
    stores/        — состояние приложения
    utils/         — утилиты (парсинг, валидация, интерполяция)
    types/         — TypeScript-типы
    constants.ts   — настраиваемые константы (INTERMEDIATE_POINTS_CNT и пр.)
    theme/         — темы (светлая/тёмная)
  ```

## ~~Шаг 2: Типы данных и константы~~ DONE
- Описать интерфейсы: `Series` (имя, данные, min/max, текущий диапазон, цвет, видимость), `DataSet` (массив серий + массив Time).
- Вынести `INTERMEDIATE_POINTS_CNT = 8` и палитру цветов в `constants.ts`.

## ~~Шаг 3: Загрузка и валидация CSV~~ DONE
- Компонент `FileLoader` — кнопка/drag-and-drop для загрузки CSV.
- Парсинг через PapaParse (`header: true`, `dynamicTyping: true`).
- Валидация:
  - Наличие колонки `Time`.
  - Значения в `Time` строго возрастают (sorted + unique).
  - При ошибке — модальное окно с описанием проблемы.
- Обработка дублирующихся имён столбцов: добавление суффикса `_<N>`.
- Сохранение результата в стор.

## Шаг 4: Стор (состояние приложения)
- Использовать React Context + useReducer (или Zustand, если будет удобнее).
- Хранить:
  - Загруженные датасеты (серии + Time-массивы).
  - Текущие диапазоны по X и по каждой Y-шкале.
  - Видимость серий (чекбоксы).
  - Текущая тема (light/dark).

## Шаг 5: Отрисовка графика (uPlot)
- Компонент `Chart` — обёртка над uPlot.
- Конфигурация:
  - Ось X — общая, диапазон из стора.
  - Для каждой серии — отдельная Y-ось (`scale`), расположенная слева.
  - Линии: `paths: uPlot.paths.linear()` (прямые).
  - Цвета из палитры, с учётом текущей темы.
- Объединение нескольких CSV в один uPlot-инстанс:
  - Так как временные шкалы из разных файлов могут не совпадать, каждая серия хранит свой собственный Time-массив. Для uPlot нужно сформировать единый aligned-массив (объединённая временная ось), а для точек, отсутствующих в данной серии, использовать `null`. uPlot нативно поддерживает `null`-значения и разрывает линию в этих местах, но рисует прямые отрезки между существующими точками.
- При суженном диапазоне X — включать ближайшие точки за границами, чтобы линии не обрывались.

## Шаг 6: Панель Y-шкал (слева от графика)
- Компонент `YAxisPanel` — список шкал для каждой серии.
- Каждая шкала показывает:
  - Название (из заголовка CSV).
  - Min и Max — кликабельные, по клику — inline-input для ввода нового значения.
  - `INTERMEDIATE_POINTS_CNT` промежуточных значений между min и max.
  - Кнопка сброса диапазона к дефолтным [dataMin, dataMax].
- Изменение диапазона через шкалу обновляет соответствующий `scale` в uPlot.

## Шаг 7: Ось X (под графиком)
- Кликабельные min/max с возможностью ввода кастомного диапазона.
- `INTERMEDIATE_POINTS_CNT` промежуточных тиков.
- Кнопка сброса.
- При изменении диапазона — перерисовка всех графиков.

## Шаг 8: Тултип по ПКМ
- Плагин uPlot или обработчик `contextmenu` на canvas.
- По ПКМ:
  - Определить позицию курсора по оси X → значение Time.
  - Для каждой видимой серии — линейная интерполяция значения в этой точке Time.
  - Показать всплывающую табличку с названиями серий и значениями.
- На позиции курсора рисовать вертикальную линию через всю область графика (crosshair) для визуальной привязки к временной оси.
- Архитектурно: логику интерполяции/расчёта значений вынести в отдельную функцию (`getTooltipValues`), чтобы её было легко заменить или сделать конфигурируемой.

## Шаг 9: Легенда с чекбоксами
- Компонент `Legend` — список серий с чекбоксами.
- По клику — переключение видимости серии (скрытие/показ линии на графике).
- Цвет маркера в легенде соответствует цвету линии.

## Шаг 10: Примечания к сессии
- Компонент `Notes` — сворачиваемая панель с `<textarea>` для свободного текста (многострочный ввод).
- Текст хранится в сторе и сохраняется в IndexedDB / экспортируется в `session.json` при экспорте `.zip`.

## Шаг 11: Тема (светлая/тёмная)
- CSS-переменные для цветов фона, текста, сетки, осей.
- Переключатель темы в хедере/тулбаре.
- uPlot: при смене темы обновлять цвета осей, сетки, фона.
- Палитра линий, подходящая под обе темы (достаточно контрастная).

## Шаг 12: Авто-сохранение сессии (IndexedDB)
- Использовать `idb-keyval` для простого key-value доступа к IndexedDB.
- При каждом значимом изменении (загрузка CSV, изменение диапазонов, переключение видимости, смена темы) сохранять полное состояние в IndexedDB:
  - Сырые данные всех серий (Time-массивы + значения).
  - Настройки: диапазоны осей, видимость, цвета, тема.
- При старте приложения:
  - Проверить наличие сохранённой сессии в IndexedDB.
  - Если есть — показать диалог «Продолжить предыдущую сессию?» / «Начать заново».
  - При выборе «Продолжить» — восстановить данные и настройки из IndexedDB.
- Дебаунс записи в IndexedDB (~500ms), чтобы не писать при каждом микроизменении.

## Шаг 13: Экспорт/импорт сессии (.zip)
- Формат — ZIP-архив (через `jszip`), содержащий:
  - `data.csv` — единый CSV, объединяющий данные всех загруженных файлов. Колонка `Time` — объединённая временная ось, для отсутствующих точек серии — пустые значения.
  - `session.json` — настройки:
    - `dataFile: "data.csv"` — путь к CSV внутри архива.
    - Диапазоны осей (X + каждая Y).
    - Видимость серий.
    - Цвета линий.
    - Тема.
- Экспорт:
  - Кнопка «Сохранить сессию» в тулбаре.
  - Формирование объединённого CSV из текущих данных.
  - Сборка ZIP через JSZip → скачивание через `URL.createObjectURL` + `<a download>`.
- Импорт:
  - Кнопка «Загрузить сессию» (или drag-and-drop файла `.zip`).
  - Распаковка ZIP через JSZip.
  - Парсинг `data.csv` через PapaParse.
  - Восстановление настроек из `session.json`.
  - Валидация структуры архива (наличие обоих файлов, корректность JSON).

## Шаг 14: Итоговая сборка и полировка
- Layout: хедер (тема, загрузка файлов, сохранение/загрузка сессии) → основная область (Y-шкалы + график + легенда).
- Обработка edge-кейсов: пустой CSV, CSV без числовых колонок, одна точка, и т.д.
- Отзывчивый resize графика при изменении окна.
